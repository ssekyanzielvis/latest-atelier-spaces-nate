## Accessing the Dashboard

### Step 1: Open Your Browser
Open your web browser and go to:
- **Production Site**: https://asn-85nt.onrender.com/admin/
- **Local Development**: http://localhost:8000/admin/

### Step 2: Login
You'll see a login screen with two fields:
- **Username**: Enter your admin username
- **Password**: Enter your admin password
- Click the **"Log in"** button

### What You'll See After Login
Once logged in, you'll land on the main dashboard showing all the sections you can manage.

## Managing Projects

### What Are Projects?
Projects are your architecture and design works that you want to showcase. Each project has details like title, description, images, client information, and location.

### Viewing All Projects

**Step 1**: Click on **"Projects"** in the sidebar or main dashboard

**What You'll See**:
- A table showing all your projects
- Columns display: Title, Category, Status, Featured, Date Created
- Buttons at the top: "Add Project", Search box, Filters

### Adding a New Project

**Step 1**: Click the **"Add Project"** button (top right)

**Step 2**: Fill in the Project Information

#### Basic Information (Required)
- **Title**: Name of the project
  - Example: "Modern Villa in Kampala"
  - Must be unique

- **Description**: Full details about the project
  - Tell the story of the project
  - Explain design choices
  - Mention challenges and solutions
  - Use the rich text editor to format (bold, italic, lists)

- **Category**: Select from dropdown
  - Choose: Residential, Commercial, Interior, Landscape, etc.
  - If category doesn't exist, you'll need to create it first

- **Main Image**: Upload the primary project photo
  - Click "Choose File"
  - Select image from your computer
  - Recommended size: 1920x1080 pixels
  - Format: JPG or PNG
  - Image will be saved to cloud storage

#### Project Details (Optional)
- **Client**: Name of the client
  - Example: "Private Client" or "ABC Corporation"

- **Location**: Where the project is located
  - Example: "Kololo, Kampala, Uganda"

- **Year**: When the project was completed
  - Just enter the year number
  - Example: 2024

- **Area**: Size of the project
  - Enter number in square meters
  - Example: 450

- **Status**: Current project status
  - Options: Completed, Ongoing, Proposed
  - Choose from dropdown

- **Featured**: Is this a featured project?
  - Check the box if you want it on homepage
  - Only check for your best 6-8 projects
  - Featured projects appear prominently on website

#### Gallery Images (Optional)
Add up to 4 additional photos:
- **Gallery Image 1**: Second project photo
- **Gallery Image 2**: Third project photo
- **Gallery Image 3**: Fourth project photo
- **Gallery Image 4**: Fifth project photo

**Tip**: Show different angles, details, and perspectives

#### System Information (Automatic)
- **Slug**: Web address name (auto-generated from title)
- **Created At**: Date added (automatic)
- **Updated At**: Last modification date (automatic)

**Step 3**: Click **"Save"** (bottom right)

**Options for Saving**:
- **Save and add another**: Save this and create another project
- **Save and continue editing**: Save but stay on this page
- **Save**: Save and return to project list

### Editing an Existing Project

**Step 1**: Go to Projects list
**Step 2**: Click on the project title you want to edit
**Step 3**: Make your changes
**Step 4**: Click "Save"

**What You Can Edit**:
- Update any field
- Replace images
- Change category
- Mark as featured/unfeatured
- Update description

### Deleting a Project

**Method 1: Delete Single Project**
1. Open the project you want to delete
2. Scroll to bottom
3. Click red "Delete" button
4. Confirm deletion when asked

**Method 2: Delete Multiple Projects**
1. Go to projects list
2. Check boxes next to projects to delete
3. Select "Delete selected projects" from dropdown menu
4. Click "Go"
5. Confirm deletion

### Filtering and Searching Projects

**Search Bar** (top right):
- Type project title, description, or client name
- Results appear instantly

**Filters** (right sidebar):
- **By Category**: Show only Residential, Commercial, etc.
- **By Status**: Show only Completed, Ongoing, or Proposed
- **By Featured**: Show only featured projects
- **By Date**: Show projects from specific date range

**Sorting**:
- Click column headers to sort
- Click again to reverse order
- Options: Title, Category, Date Created

## Managing News Articles

### What Are News Articles?
News articles are blog posts, announcements, or updates you want to share with visitors. They appear in the news section of your website.

### Viewing All News Articles

**Step 1**: Click on **"News Articles"** in the sidebar

**What You'll See**:
- Table of all articles
- Columns: Title, Author, Published Date, Featured
- Search and filter options

### Adding a New Article

**Step 1**: Click **"Add News Article"** button

**Step 2**: Fill in Article Information

#### Required Fields
- **Title**: Article headline
  - Example: "New Project Launch in Nakasero"
  - Keep it catchy and clear

- **Content**: Main article text
  - Full story or announcement
  - Use the editor to format
  - Add paragraphs, headings, lists
  - Can be several paragraphs long

- **Image**: Article cover photo
  - Upload a relevant image
  - This appears at top of article
  - Recommended: 1200x630 pixels

#### Optional Fields
- **Excerpt**: Short summary (150-200 characters)
  - Brief teaser about the article
  - Shown in article previews
  - If left empty, automatically generated from content

- **Author**: Who wrote the article
  - Your name or team member's name
  - Example: "Nate Ssekyanzi"

- **Published Date**: When article goes live
  - Defaults to today
  - You can schedule for future date
  - Format: Day/Month/Year

- **Featured**: Mark as important
  - Check if this should appear on homepage
  - Only feature your most important news

#### System Fields (Automatic)
- **Slug**: Web address (from title)
- **Created At**: Date added
- **Updated At**: Last edited

**Step 3**: Click **"Save"**

### Editing an Article

**Step 1**: Find article in list
**Step 2**: Click article title
**Step 3**: Make changes
**Step 4**: Save

**Common Edits**:
- Fix typos
- Update information
- Add more details
- Change featured status
- Replace image

### Deleting Articles

Same process as deleting projects:
- Single: Open and click Delete button
- Multiple: Select and choose "Delete selected"

### Publishing Schedule

**Draft vs Published**:
- All articles are published immediately by default
- To "hide" an article: delete it or set future published date

**Scheduling**:
1. Set Published Date to future date
2. Save article
3. It will only show on website after that date

## Managing Team Members

### What Are Team Members?
These are the people profiles shown on your "Team" page - architects, designers, and staff.

### Viewing Team Members

**Step 1**: Click **"Team Members"** in sidebar

**What You'll See**:
- List of all team members
- Columns: Name, Position, Active Status, Order
- Each person's photo (thumbnail)

### Adding a New Team Member

**Step 1**: Click **"Add Team Member"**

**Step 2**: Enter Team Member Details

#### Required Information
- **Name**: Person's full name
  - Example: "Nate Ssekyanzi"

- **Position**: Job title or role
  - Example: "Lead Architect"
  - Example: "Interior Designer"
  - Example: "Project Manager"

- **Photo**: Professional headshot
  - Upload clear photo of person
  - Square image works best (500x500)
  - White or neutral background preferred

#### Optional Information
- **Bio**: About the person
  - Education background
  - Experience and specializations
  - Achievements
  - Professional philosophy
  - 2-3 paragraphs

- **Email**: Contact email address
  - Will be shown on website
  - Example: nate@atelierspacesnate.com

- **Phone**: Contact phone number
  - Include country code
  - Example: +256 700 000000

- **LinkedIn**: LinkedIn profile URL
  - Example: https://linkedin.com/in/natessekyanzi
  - Full web address including "https://"

- **Twitter**: Twitter profile URL
  - Example: https://twitter.com/natessekyanzi
  - Include full address

- **Order Position**: Display order (number)
  - Lower numbers appear first
  - Example: 1 for first person, 2 for second
  - Use this to control team display order

- **Is Active**: Show on website?
  - Check to display this person
  - Uncheck to hide (without deleting)
  - Useful for temporary staff or leave

#### System Fields
- **Created At**: When added
- **Updated At**: Last modified

**Step 3**: Click **"Save"**

### Editing Team Members

**Step 1**: Click on person's name
**Step 2**: Update information
**Step 3**: Save

**Common Updates**:
- New photo
- Updated position/title
- New contact details
- Updated bio
- Change display order

### Reordering Team Members

**Step 1**: Edit each team member
**Step 2**: Set **Order Position** number
**Step 3**: Save

**Example Order**:
- Founder/Lead: Order = 1
- Senior Architect: Order = 2
- Designer: Order = 3
- Intern: Order = 4

**Result**: Team displays in this order on website

### Hiding vs Deleting

**To Temporarily Hide**:
- Edit team member
- Uncheck "Is Active"
- Save
- They won't show on website but data remains

**To Permanently Delete**:
- Open team member
- Click Delete button
- Confirm
- All data removed

## Managing Hero Slides

### What Are Hero Slides?
Hero slides are the large, full-width images that appear at the top of your homepage, cycling automatically. They're the first thing visitors see.

### Viewing Hero Slides

**Step 1**: Click **"Hero Slides"** in sidebar

**What You'll See**:
- List of all slides
- Columns: Title, Subtitle, Order, Active Status
- Preview thumbnails

### Adding a New Hero Slide

**Step 1**: Click **"Add Hero Slide"**

**Step 2**: Create Your Slide

#### Required Fields
- **Title**: Main headline on slide
  - Large text shown on image
  - Example: "Innovative Architectural Design"
  - Example: "Creating Spaces That Inspire"
  - Keep it short (3-6 words)

- **Image**: Background photo
  - Upload high-quality image
  - Recommended: 1920x1080 or larger
  - Use striking architecture photos
  - Image will fill entire screen width

#### Optional Fields
- **Subtitle**: Supporting text
  - Appears below title
  - Additional context or tagline
  - Example: "Blending form with function"
  - 1-2 short sentences

- **Call to Action Text**: Button text
  - What the button says
  - Example: "View Our Work"
  - Example: "Contact Us"
  - Example: "Explore Projects"
  - Leave empty if no button wanted

- **Call to Action Link**: Where button goes
  - Web address button links to
  - Example: "/projects"
  - Example: "/collaborate"
  - Must start with "/" for internal links

- **Order Position**: Slide sequence (number)
  - Lower numbers show first
  - Example: 1 for first slide, 2 for second
  - Slides cycle in this order

- **Is Active**: Show this slide?
  - Check to display in rotation
  - Uncheck to hide temporarily
  - Only active slides appear on website

**Step 3**: Click **"Save"**

### Slide Management Tips

**How Many Slides?**
- Recommended: 3-5 slides
- Minimum: 1 slide required
- Maximum: No limit but 5 is optimal
- Too many = visitors don't see them all

**What Makes a Good Hero Slide?**
- High-resolution image
- Clear, impactful title
- Not too much text
- Strong visual composition
- Represents your brand well

**Automatic Cycling**:
- Slides change every 5 seconds
- Visitors can manually navigate
- Cycles continuously
- Only active slides included

### Editing Hero Slides

**Step 1**: Click slide title
**Step 2**: Make changes
**Step 3**: Save

**Common Edits**:
- Replace image with better photo
- Update text for clarity
- Change order to prioritize
- Activate/deactivate temporarily
- Update call-to-action

## Managing About & Slogan Sections

These are special sections - only ONE of each can exist. They control specific parts of your homepage.

### About Section

**What It Is**:
The "About Us" content that appears on your homepage and about page.

**Step 1**: Click **"About Section"** in sidebar

**What You'll See**:
- If exists: The current about section (click to edit)
- If empty: Option to add first about section

**Step 2**: Edit About Section Content

#### Fields Available
- **Title**: Main heading
  - Example: "About Atelier Spaces Nate"
  - Example: "Who We Are"

- **Content**: Main about text
  - Your studio's story
  - Philosophy and approach
  - What makes you unique
  - Several paragraphs
  - Use rich text formatting

- **Mission**: Mission statement (optional)
  - Your core purpose
  - What you aim to achieve
  - 1-2 sentences

- **Vision**: Vision statement (optional)
  - Your future goals
  - Where you're heading
  - 1-2 sentences

- **Values**: Core values (optional)
  - What you stand for
  - Guiding principles
  - Can be bullet points

- **Image**: About section photo (optional)
  - Team photo
  - Studio photo
  - Project photo
  - Recommended: 800x600

**Step 3**: Click **"Save"**

**Important Note**:
- Only ONE about section exists
- Don't create multiple
- Edit the existing one to update

### Slogan Section

**What It Is**:
The prominent banner section on your homepage with a main slogan and sub-slogan.

**Step 1**: Click **"Slogan Section"** in sidebar

**What You'll See**:
- Current slogan (if exists)
- Option to create first slogan

**Step 2**: Edit Slogan Content

#### Fields Available
- **Main Slogan**: Primary headline
  - Bold, impactful statement
  - Example: "Design That Transforms Spaces"
  - Example: "Where Innovation Meets Tradition"
  - 3-8 words recommended
  - Shows large on homepage

- **Sub Slogan**: Supporting text
  - Elaborates on main slogan
  - Example: "Creating architectural excellence across Uganda"
  - 1-2 sentences
  - Shows smaller below main slogan

- **Background Image**: Banner photo (optional)
  - Full-width background
  - Should work with text overlay
  - Use image with neutral areas for text
  - 1920x600 recommended

**Step 3**: Click **"Save"**

**Important Note**:
- Only ONE slogan section exists
- Edit to update, don't create multiples

**What Visitors See**:
A prominent banner on homepage displaying your slogan over the background image.

## Viewing Collaboration Requests

### What Are They?
When visitors fill out your "Collaborate" or "Contact" form on the website, their submissions appear here.

**Step 1**: Click **"Collaborations"** in sidebar

**What You'll See**:
- Table of all submissions
- Columns: Name, Email, Company, Project Type, Status, Date
- Most recent at top

### Viewing a Submission

**Step 1**: Click on person's name

**What You'll See**:
- **Name**: Person who submitted
- **Email**: Their email address
- **Company**: Their company (if provided)
- **Phone**: Contact number (if provided)
- **Project Type**: Type of project they're interested in
- **Budget**: Budget range (if provided)
- **Message**: Their full message
- **Status**: Current status (Pending, Contacted, Completed)
- **Created At**: When they submitted

### Managing Submissions

#### Changing Status

**Step 1**: Open the submission
**Step 2**: Find **Status** dropdown
**Step 3**: Select new status:
- **Pending**: Not yet contacted (default)
- **Contacted**: You've reached out
- **Completed**: Project done or declined

**Step 4**: Click **Save**

**Why Change Status?**
- Track which leads you've followed up
- Know what's pending action
- Keep organized

#### Taking Action

**Contact the Person**:
- Copy their email
- Copy their phone
- Use information to reach out
- Update status after contacting

**Make Notes**:
- Unfortunately, no notes field built in
- Use external system for detailed tracking
- Status helps basic organization

#### Deleting Submissions

**Single Deletion**:
- Open submission
- Click Delete button
- Confirm

**Multiple Deletion**:
- Check boxes in list
- Select "Delete selected"
- Click Go
- Confirm

**When to Delete**:
- Spam submissions
- Completed projects (after archiving info)
- Old pending requests (after follow-up attempts)

### Filtering Collaborations

**By Status**:
- Right sidebar filter
- Show only Pending, Contacted, or Completed

**By Date**:
- Filter by submission date
- Find old or recent requests

**Search**:
- Search by name, email, company
- Find specific person

### Best Practices for Collaborations

1. **Check Daily**: Review new submissions regularly
2. **Respond Quickly**: Update status after each action
3. **Be Thorough**: Read full message before responding
4. **Track Externally**: Use email or CRM for detailed tracking
5. **Clean Up**: Delete spam and old completed requests

## Tips & Best Practices

### General Dashboard Usage

#### Working Efficiently

**Keyboard Shortcuts**:
- Press **Tab** to move between fields
- Press **Enter** to submit forms
- Press **Esc** to close popups

**Quick Navigation**:
- Use sidebar for section jumping
- Use browser back button to go back
- Bookmark admin URL for quick access

**Saving Your Work**:
- Save frequently while editing
- Use "Save and continue editing" to keep working
- Don't navigate away without saving

#### Managing Images

**Image Best Practices**:
1. **Resize Before Upload**
   - Don't upload huge files (over 5MB)
   - Resize to recommended dimensions
   - Reduces upload time and storage

2. **Name Files Descriptively**
   - Bad: IMG_1234.jpg
   - Good: villa-exterior-kampala.jpg
   - Helps organization

3. **Use High Quality**
   - Clear, sharp photos
   - Good lighting
   - Professional looking

4. **Consistent Style**
   - Similar editing style
   - Consistent dimensions
   - Same orientation when possible

**Image Dimensions Guide**:
- **Hero Slides**: 1920x1080 (landscape)
- **Projects**: 1200x800 (landscape)
- **News**: 1200x630 (landscape)
- **Team**: 500x500 (square)
- **Works**: 1200x1200 (square)
- **About/Slogan**: 1600x900 (landscape)

#### Content Writing Tips

**Keep It Professional**:
- Check spelling and grammar
- Read aloud to check flow
- Be clear and concise
- Avoid jargon unless necessary

**SEO-Friendly Content**:
- Use descriptive titles
- Include relevant details
- Use proper headings
- Keep paragraphs short

**Consistent Tone**:
- Match your brand voice
- Be authentic
- Professional but approachable
- Consistent across all content

### Organization Strategies

#### Featured Content

**What to Feature**:
- Your absolute best work
- Recent completed projects
- Prestigious clients
- Award-winning designs
- Most impressive visually

**How Many to Feature?**
- Projects: 6-8 maximum
- News: 3-4 maximum
- Works: 4-6 maximum
- Too many = nothing stands out

#### Categorization

**Categories Best Practices**:
- Not too many (5-10 is good)
- Clear, distinct categories
- No overlap between categories
- Easy for visitors to understand

#### Ordering Content

**Use Order Numbers**:
- Team members: By seniority
- Hero slides: Best first
- Priority content gets lower numbers
- Reorder as needed

### Security & Maintenance

#### Password Security

**Create Strong Passwords**:
- At least 12 characters
- Mix of letters, numbers, symbols
- Not your name or "password"
- Change regularly

**Changing Your Password**:
1. Click "Change Password" (top right)
2. Enter current password
3. Enter new password twice
4. Click "Change Password"

#### Regular Maintenance

**Weekly Tasks**:
- Check for new collaboration requests
- Review recent changes (Recent Actions)
- Add any new news or projects

**Monthly Tasks**:
- Review all content for accuracy
- Update team member info if changed
- Check all images load properly
- Delete old/irrelevant content

**Quarterly Tasks**:
- Review featured content selection
- Update about section if needed
- Refresh hero slides with new images
- Audit and organize categories

### Getting Help

#### If Something Goes Wrong

**Changes Not Showing?**:
- Clear your browser cache
- Try different browser
- Check you clicked Save
- Refresh the website

**Can't Upload Image?**:
- Check file size (under 5MB)
- Check file format (JPG or PNG)
- Try different image
- Compress image first

**Forgot Password?**:
- Use "Forgot Password" link
- Or contact website administrator
- May need database access to reset

**Content Looks Wrong?**:
- Check all required fields filled
- Verify image dimensions
- Check for typos in URLs
- Preview before saving

#### Contact Support

If you need technical help:
- Note what you were doing
- Note any error messages
- Take screenshots
- Contact your web developer

### Common Questions

**Q: How often should I update content?**
A: Add news monthly, update projects as completed, keep team current.

**Q: Can I schedule content to publish later?**
A: Yes, for news articles - set future published date.

**Q: What happens to deleted items?**
A: They're permanently removed. Can't be recovered without database backup.

**Q: Can multiple admins work at once?**
A: Yes, but avoid editing same item simultaneously.

**Q: How do I preview changes before publishing?**
A: Save, then click "View Site" to see live site.

**Q: Can I restore previous versions?**
A: Not built-in. Make backups before major changes.
